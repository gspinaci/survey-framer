QUICK START
================================================================================

1. SETUP (5 min)
   python3 -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt
   cp .env.example .env
   # Edit .env, add HARVARD_BEDROCK_API_KEY from portal.apis.huit.harvard.edu

2. PREPARE INPUT (10 min)
   Create input/papers.csv with your papers
   See CSV_GUIDE.md for format

3. RUN (90-150 min for 30 papers)
   python main.py --input input/papers.csv

4. REVIEW OUTPUTS
   output/papers/                (per-paper reports)
   output/narrative_synthesis.md (final narrative)

================================================================================
FILE PURPOSES
================================================================================

main.py
  Purpose: CLI entry point and orchestrator
  Usage:   python main.py --input input/papers.csv [--step fetch|analyze|narrative|all]
  Tasks:   Coordinates all 6 pipeline steps with progress bars

config.py
  Purpose: Centralized configuration
  Contains: API endpoints, paths, model IDs, email for DOI resolution
  Edit:    Only change API_KEY, UNPAYWALL_EMAIL if needed

csv_reader.py
  Purpose: Parse input CSV and extract paper identifiers
  Extracts: arxiv_id, doi from URLs using regex
  Output:  List of paper dicts with all CSV fields

pdf_fetcher.py
  Purpose: Download PDFs from multiple sources
  Strategy: arXiv → DOI/Unpaywall → Direct URL → Skip
  Output:  Path to saved PDF or None

pdf_extractor.py
  Purpose: Extract plain text from PDF files
  Limits:  200k tokens max (~80 pages)
  Handles: Gracefully skips scanned PDFs

analyzer.py
  Purpose: Send paper text to Claude via Harvard Bedrock API
  Extracts: benchmark_name, modalities, analytical_tasks, inclusion_criteria
  Output:  Structured JSON with 20+ fields

report_writer.py
  Purpose: Generate per-paper markdown and JSON reports
  Format:  Table of inclusion criteria, modality/task details
  Output:  {paper_id}.md and {paper_id}.json

narrative.py
  Purpose: Synthesize all analyses into narrative survey
  Generates: Thematic clusters, modality landscape, framing options, gaps
  Output:  narrative_synthesis.md

requirements.txt
  Purpose: Python dependencies
  Install:  pip install -r requirements.txt
  Packages: requests (API), PyMuPDF (PDF extraction), python-dotenv (env vars), tqdm (progress)

.env.example
  Purpose: Template for environment variables
  Usage:   Copy to .env, fill in HARVARD_BEDROCK_API_KEY
  Security: .env is gitignored (never commit your key!)

KEY FEATURES (NEW)
================================================================================

✨ Benchmark Modalities Extraction
   - Identifies: text, image, audio, video, multimodal
   - Explains: How modalities are combined
   - Used in: Per-paper reports + narrative landscape analysis

✨ Analytical Task Type Identification
   - Identifies: Factual recognition, causal reasoning, generation, numeric reasoning, etc.
   - Explains: Detailed nature and complexity of tasks
   - Used in: Per-paper reports + narrative gap analysis

✨ Modality & Task Landscape Analysis
   - Coverage: Which modality combinations are well-covered?
   - Dominance: Which analytical tasks appear most?
   - Gaps: Missing combinations and underrepresented types

✨ Enhanced Narrative Synthesis
   - Thematic clustering by modality, task type, ASJC area
   - Landscape overview (what's covered, what's missing)
   - Framing options for different audiences
