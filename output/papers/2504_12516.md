## Benchmark Modalities

**Modality Types:** text

**Description:** The benchmark consists entirely of short-form text natural language questions and textual reference answers, requiring agents to navigate text-based search queries and web pages.

## Analytical Tasks

**Task Types:** fact retrieval, multi-hop reasoning, constraint satisfaction, fact verification

**Detailed Description:** Tasks require agents to resolve complex, multi-constraint queries (e.g., matching a fictional character to specific backstory criteria) by dynamically searching, parsing multi-hop constraints, and cross-referencing information across the open web to locate a single, verifiable short-string answer.

## Summary

BrowseComp is a general-purpose benchmark comprising 1,266 challenging, short-answer questions designed to evaluate the capability of web browsing agents to persistently search and retrieve obscure, deeply entangled information. The dataset features 'inverted questions' that are extremely difficult for humans or standard LLMs to solve but simple to verify automatically. The evaluation demonstrates that while standard LLMs fail near-completely, agentic systems utilizing search and reinforcement learning (like OpenAI Deep Research) can achieve high accuracy that scales with test-time compute.

## Key Findings

Standard models without browsing fail on this obscure fact retrieval, with GPT-4o achieving 0.6% and GPT-4.5 0.9% accuracy. OpenAI o1 achieves 9.9% utilizing internal reasoning alone. OpenAI Deep Research achieves 51.5% accuracy, which scales smoothly with increased test-time compute, reaching over 70% accuracy when utilizing parallel sampling and a best-of-N voting scheme. In contrast, human trainers solved only 29.2% of the tasks within a two-hour limit.
